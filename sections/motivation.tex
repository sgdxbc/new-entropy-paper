\subsection{System Model and Goal}
\label{sec:motivation:model}

We assume multiple permissionless nodes contributing storage resources in DSNs, and the nodes follow a hybrid security model.
Among these nodes, we assume up to a certain fraction (e.g., \nicefrac{1}{3}) are Byzantine and the other nodes are rational~\cite{bar}.
The Byzantine nodes may arbitrarily diverge from the expected behavior even if that would cause economic loss on them.
They are potentially malicious, attacking the system and attempting to cause any data loss.
At the same time, the rational nodes try to maximize their profits.
We assume there exists an economic incentive mechanism, which can be employed by DSNs to effectively drive rational nodes to perform certain action.
For example, in \sys nodes get reward for transmitting ``useful'' bits to clients, and we assume the value of reward is higher than the cost of persisting and transfer the bits, so rational nodes are incentivized for correct bit transmissions.
We will detail the prototype incentive design in \XXX.

The rational nodes will only follow the protocol if that is the most profitable choice.
For example, they may discard data copies that they are responsible to store if they believe that they won't be caught and penalized for this.
So, DSNs must be able to verify the behaviors they want to incentivize, such as a common practice of employing proof of retrievability~\cite{por} to verify correct storage.
At the same time, the rational nodes experience benign failures at a certain rate.
They unintentionally lose stored data for various reasons, including hardware wore out, catastrophic events and simply churned from the systems.
These failures can be significant in heterogeneous storage, especially the resource-constraint devices.
We define the faulty rate $f$ as the maximum fraction of the Byzantine faulty nodes and the failure rate $f_r$ as the average failed fraction of the rational nodes during a certain time interval $T$.

Under this model, DSNs must keep data available while tolerate both $fN$ faulty nodes and $f_r(1-f)N$ failures from rational nodes happened in $T$, where $N$ is the total number of nodes.
Tolerating the faulty nodes is more challenging than tolerating innocent failures, because the faulty nodes may strategically attack the systems.
For example, they may attempt to pick the data they store e.g., by picking specific node IDs, collude with each other and create correlated failures when that will lead to data loss.
We assume Sybil attack can be alleviated (i.e., assuming a known $f$) by requiring nodes to stake to be qualified to participate.
DSNs provides two APIs: \texttt{put} to store data with expiration and obtain a reference, and \texttt{get} to retrieve data with the reference.
In all DSNs studied in this work, the data hashes are used as the references, so the retrieved data can be easily validated with the references.
Data availability means that between \texttt{put} returns and the data expired, \texttt{get} can return the valid data successfully.

\sys supports varying number of nodes $N$.
For simplicity, we treat $N$ of each time interval $T$ as constant and equals to the minimum number of nodes during $T$.
We assume $N$ for every time interval is known.
